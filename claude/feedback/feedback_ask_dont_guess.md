---
name: Ask don't guess
description: User prefers to be asked clarifying questions rather than having Claude make assumptions
type: feedback
---

When uncertain about what the user means or intends, ask them directly rather than making assumptions and proceeding.

**Why:** Spending time reasoning through multiple interpretations in my own head, then implementing one and noting caveats, is worse than a single upfront question. The user has to read the caveats anyway — better to have just asked.

**How to apply:** If a request has a genuine ambiguity that affects implementation (e.g. a colour that would be invisible on its own background), stop and ask before writing any code.
